# Software Developer — Career-Ladder Scope

**Scope tier:** mid (career ladder). Sits between the [industry-level view](../Software%20Industry/) (widest) and the [CTO case study](../CTO/) (sharpest).

**Core claim:** In most professions seniority *deepens* the craft. In software, the ladder's one institutionally-real discontinuity — the **IC→management rotation** the industry split into two separate career tracks — is a default advancement step, surrounded by a recurring pattern of further rotations (Senior→Staff most clearly) that the title structure labels as natural "deepening." Each is a move to a **distinct skill set the ladder gives no on-the-job runway to acquire** — not an *incompatible* one (the engineer/manager pendulum shows positive transfer). The CTO composite problem is the terminal expression of a mismatch that begins at the first management-track promotion.

> *"What we have here is a human capacity issue, disguised as a 'smart people should be able to learn everything' problem."*
