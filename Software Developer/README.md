# Software Developer — Career-Ladder Scope

**Scope tier:** mid (career ladder). Sits between the [industry-level view](../Software%20Industry/) (widest) and the [CTO case study](../CTO/) (sharpest).

**Core claim:** In most professions seniority *deepens* the craft. In software, seniority *abandons* it — every promotion (Junior → Mid → Senior → Staff/Principal → EM → Director → VP) is a rotation to a different, often incompatible skill set. The CTO composite problem is the terminal expression of a mismatch that begins at the **first** promotion.

> *"What we have here is a human capacity issue, disguised as a 'smart people should be able to learn everything' problem."*
