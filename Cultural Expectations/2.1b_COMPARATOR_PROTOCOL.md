---

## 1. The question, stated so it can be answered

**Not:** "Do CFO/CMO/COO JDs contain the CTO's Builder/Multiplier/Strategist modes?" — that is rigged
(the bins are CTO-shaped; G-3 circularity, R1 Lead 3).

**Instead, frame-neutral, asked identically of every role:**

> Does the JD demand a **composite of the role's *own* conflicting modes** — deep functional craft
> **and** people leadership **and** external/board strategy, held **simultaneously and unbounded** —
> or does it present a **bounded role** (one or two modes, or explicit scope limits that resolve the
> tension)?

Every C-suite role has its own three-mode analogue. Coding each role in *its own* modes is the only
fair test:

| Mode (generic) | CTO | CFO | CMO | COO |
|---|---|---|---|---|
| **Craft / hands-on** | writes/architects code | builds models, closes books, technical accounting | runs campaigns, owns the funnel/creative | designs process, runs operations hands-on |
| **People / function-leader** | leads eng org | leads finance/accounting team | leads marketing org | leads cross-functional org |
| **Strategy / board-facing** | tech strategy, board | capital strategy, fundraising, board | brand/growth strategy, board | company operating strategy, board |

---

## 2. The four scores per JD (identical for all roles)

For each JD, record:

1. **Modes present (0–3):** how many of the role's own three modes the JD demands as load-bearing.
2. **Bounded? (Y/N):** does the JD use scope-limiting language that *resolves* the tension (e.g.
   "you will not be hands-on," "this is a pure strategy role"), or only language that excludes an
   extreme while keeping ≥2 modes in tension (the CTO pattern)?
3. **Tension named? (Y/N):** does the JD itself name the trade-off between modes (vs. naming it only
   in commentary, or not at all)? — the T2-3 legibility test, applied to comparators.
4. **Title carries scope? (Y/N):** does the title specify which modes (e.g. "VP Finance, Controller
   track") or is it a bare composite signal ("CFO") — the Axis-8 test.

Score the comparators with the **same rubric and the same coder discipline** used on the 7 CTO JDs in
`../DO NOT CHECK THIS IN/JD_audit_synthesis_wellfound_20260626.md`.

---

## 3. The decision rule (pre-registered — fixed before the data is seen)

Pre-committing the verdict map so the result can't be narrated after the fact (the R2 confirmation-
bias guard):

| Observation across the 9 comparator JDs | Verdict on G-1 |
|---|---|
| **CFO/CMO/COO are mostly composite (≥2 modes, unbounded), like the CTO** | **CTO-specificity is FALSE.** "Composite" is a **C-suite-wide** framing. T2 must be restated as "C-suite hiring artifacts apply a context-free composite standard" — *not* CTO-specific. (Draft A already hedges to this; this would confirm the hedge was correct.) |
| **CFO/CMO are mostly bounded but COO is composite** | **CTO-specificity is PARTIALLY SUPPORTED.** The composite tracks *breadth-of-mandate roles* (CTO, COO), not the CTO alone. T2 narrows to "broad-mandate C-suite roles," with the CTO as one instance. |
| **CFO/CMO/COO are mostly bounded; only the CTO is composite** | **CTO-specificity SURVIVES.** Something about the CTO (or tech roles) specifically attracts the composite framing. G-1 clears; T2's CTO-specific form is earned. |
| **COO scores bounded** (the control fails) | **METHOD BROKEN — do not report a verdict.** COO is a known catch-all; if the rubric can't detect it as composite, the rubric is under-counting and the whole comparison is invalid. Re-examine coding before any G-1 claim. |

**Stage interaction (secondary):** if composite-ness in comparators is stage-modulated the same way
the CTO's is (early excludes one extreme, growth excludes another, composite always retained between),
that is *additional* evidence the framing is structural and C-suite-wide, not CTO-specific.

## 5. Honest limits of this test (stated up front, per the corpus's own discipline)

- **Still Wellfound-only, still content-site source class.** This closes the *comparator* gap, not the
  *peer-review* gap (Draft A §8 gap #3 remains open). It tests CTO-specificity, not whether JDs predict
  real role failure.
- **n=3 per role** detects a gross composite-vs-bounded difference and a stage pattern; it does **not**
  give a tight rate. If the result is close (e.g. CFO splits 2-composite/1-bounded), that is a
  "needs more samples" outcome, not a verdict — say so rather than forcing a call.
- **Coder is the same instance that built the corpus.** Mitigation: the §3 decision rule is
  pre-registered above, and the COO control catches under-counting in the CTO-favoring direction.
