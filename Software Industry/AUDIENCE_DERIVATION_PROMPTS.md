# Audience Derivation Prompts — 6 Chats, One Per Actor

**Purpose:** Six separate chats, each producing an audience-specific version of the public draft from the *same* rigorous spine. Hand one prompt per chat. Each derived draft is a **projection** of `DRAFT_SYNTHESIS.md` (softening downward, losslessly) — **not a new argument.**

**This is the widest (industry-structural) tier.** Its derivations inherit the rotation/cushion/misattribution machinery the sibling tiers also use, but each one must carry *this* tier's distinctive payload: the **specialization frame** (every mature profession specialized at the capacity ceiling; software rotated instead) and the **fungibility engine** (§0.2 — the root cause that makes a rotating ladder look reasonable). If a derived draft reads like it could have come from the [Software Developer](../Software%20Developer/) or [CTO](../CTO/) tier, it has lost its altitude — pull it back up to the cross-industry comparison.

**Hard rule for all six (state it in every chat):** the rigorous spine `DRAFT_SYNTHESIS.md` is the **single source of truth**. A derived draft may *re-weight, re-voice, and drop* — it may **never add a claim not in the spine, nor restore a claim the spine's §6 concessions or the must-not-claim table retired.** If a derived draft needs a claim the spine doesn't license, that's a finding to bring *back* to the spine, not to assert in the derivation.

**Shared inputs (every chat reads these first):**
- `Software Industry/DRAFT_SYNTHESIS.md` — the spine (§0–§6) + the **Audience Map** (each actor's column is that draft's brief)
- `Software Industry/R3_STEELMAN_strongest_form.md` — the must-not-claim table + the one-paragraph version + the five concessions
- `Software Industry/R2_FALSIFICATION_and_bias_audit.md` — the falsification protocol and the seven bias failure-modes (so the derivation doesn't reintroduce a tell the spine already cooled)
- `Software Industry/08_CITATIONS_gap_closure.md` — what is verified to primary (imposter base rates, ISBSG, DORA/SPACE) vs. still illustrative, so a derivation doesn't over-state a citation

**Borrowed-evidence boundary (this tier's equivalent of the CTO's Premise 1):** the peer-reviewed identity-transition mechanism (Mathias & Williams 2018, Van Lancker et al. 2023) is established **in founders**, not in engineers; the engineer/IC application is *this corpus's extension*, argued not borrowed. State it once, lightly, in every derivation — never let a derived draft imply the founder studies measured engineers.

**Shared output spec:** `Software Industry/derived/DRAFT_SYNTHESIS_<actor>.md`. Commit when done. Keep the audience-map row for that actor at the top as the brief. Carry a one-line "derived from `DRAFT_SYNTHESIS` @ <commit>" provenance stamp.

## After all six

Each derived draft should be auditable against the spine the same way the increments were: a derived claim that isn't traceable to a spine section (or that restores a retired claim) is a defect. A quick post-pass: grep each derived draft for the must-not-claim left-column phrases and verify any hit is a *denial*, not a regression:

- **"incompatible"** — must appear only as a denial ("not incompatible — distinct, no runway"). Any bare assertion is a regression (this was retired corpus-wide; see the README reconciliation note).
- **"no stable craft"** — must be the narrowed form ("the *credited* layer is unstable; the durable layer is uncredited").
- **"uniquely broken"** — only as "behavioral outcomes match no comparator's cushions," never as "software is more broken / just more surveyed" left un-rebutted.
- **"caused by"** on imposter syndrome — must be "candidate explanation never tested," never causal.
- **"avoids accountability" / intent language** — must be "structure that *lets* failures be read as individual."
- **"specializes when affordable"** (the §5↔§6c7 trap) — must be "cushions/distributes the *composite* when affordable," never "fully specializes the leadership rungs when affordable."
- **CHAOS** figures used as load-bearing — should be ISBSG; CHAOS illustrative only.

Any hit that isn't a denial or the narrowed form is a regression to fix.
